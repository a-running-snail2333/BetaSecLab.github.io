    提权：提升操作系统权限；此篇文章主要是介绍系统提权的常规方法，提权方式在渗透实战中不是必须的过程。但是当需要抓取口令，删除系统软件，权限限制查看敏感文件等情况下，就需要进行权限提升。此篇文章总结整理了两大操作系统：linux系统和windows系统下常规提权方式。文章存在不足或错误请尽快与本人联系。

    0x00 windows系统下的提权

    windows操作系统下的权限包括：操作系统内核提权、凭证存储、

    1) windows系统内核提权

    在经过常规手法web渗透或者钓鱼等操作获得基本权限(用户权限，或者服务权限)之后，最常用的方法就是windows系统的内核提权。这种方法是最普遍、最简单、最容易达到预期目标；
    查看系统补丁:
    systeminfo //cmd下执行可以查看系统版本，主机名，打补丁情况等

    wmic qfe get Caption,Description,HotFixID,InstalledOn //wmic方式查询系统补丁 wmic qfe list

    use post/windows/gather/enum_patches //msf下模块

    Powershell Get-HotFix //powershell脚本

    查询缺失补丁:
    辅助工具： Windows-Exploit-Suggester： https://github.com/GDSSecurity/Windows-Exploit-Suggester/
    首先收集sysinfo信息到.txt文件，并命名位systeminfo.txt。通过py脚本逐行调用并与微软公开的CVE库匹对，查看系统打补丁情况，并列出未打补丁的kb号以及对应可利用的exp地址；
    首先更新库： python Windows-Exploit-Suggester.py -u
    其次执行脚本： python windows-exploit-suggester.py –database 2018-12-03-mssb.xls –systeminfo systeminfo.txt

    2）凭证存储

    通常情况下，当一名黑客已经获取某个系统访问权想要进一步提升权限时，首先要做的第一步就是要搜索整个系统，探测是否存在本地管理员账户以及密码。现对于其他类型的提权方式，此方法更不容易被管理员发现。
    windows系统配置：
    管理员通常使用Windows系统进行部署服务，通过网络将该镜像部署到各种系统中，称为无人值守安装。此类方法进行安装存在的问题主要是，本地管理员密码以明文或Base-64编码的形式存储在不同的位置。

    C:\unattend.xml
C:\Windows\Panther\Unattend.xml
C:\Windows\Panther\Unattend\Unattend.xml
C:\Windows\system32\sysprep.inf
C:\Windows\system32\sysprep\sysprep.xml

    具体路径大体了解即可，通常在实战中可以通过相关工具进行搜索：
    metasploit模块：post/windows/gather/enum_unattend
    PowerSploit脚本：

    Get-UnattendedInstallFile
Get-Webconfig
Get-ApplicationHost
Get-SiteListPassword
Get-CachedGPPPassword
Get-RegistryAutoLogon
