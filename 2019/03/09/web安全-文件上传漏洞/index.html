    文件上传漏洞可以说是日常渗透测试用得最多的一个漏洞，因为用它获得服务器权限最快最直接。但是想真正把这个漏洞利用好却不那么容易，其中有很多技巧，也有很多需要掌握的知识。俗话说，知己知彼方能百战不殆，因此想要研究怎么防护漏洞，就要了解怎么去利用。

<script type="text/javascript">
    function checkFile() {
        var file = document.getElementsByName('upload_file')[0].value;
        if (file == null || file == "") {
            alert("请选择要上传的文件!");
            return false;
        }
        //定义允许上传的文件类型
        var allow_ext = ".jpg|.png|.gif";
        //提取上传文件的类型
        var ext_name = file.substring(file.lastIndexOf("."));
        //判断上传文件类型是否允许上传
        if (allow_ext.indexOf(ext_name) == -1) {
            var errMsg = "该文件不允许上传，请上传" + allow_ext + "类型的文件,当前文件类型为：" + ext_name;
            alert(errMsg);
            return false;
        }
    }
</script>

    对以上代码进行讲解：
    html部分：
    <h3> ：标签标题
    <form> ：标签用于为用户输入创建 HTML 表单，具体的属性值此处不做详细讲解，唯独注意的是文件上传需要将enctype属性设置为multipart/form-data。
    <input> ：标签用于输入标签，type属性规定输入的类型。
    php部分：
    isset：检测变量是否设置。如果 var 存在并且值不是 NULL 则返回 TRUE，否则返回 FALSE
    file_exists：函数检查文件或目录是否存在。如果指定的文件或目录存在则返回 TRUE，否则返回 FALSE。
    move_uploaded_file()：函数把上传的文件移动到新位置。如果成功该函数返回 TRUE，如果失败则返回 FALSE。
    $_files：你可以从客户计算机向远程服务器上传文件。第一个参数是表单的name，第二个下标可以是name、type、size、tmp_name 或 error

    $_FILES["file"]["name"] - 被上传文件的名称
$_FILES["file"]["type"] - 被上传文件的类型
$_FILES["file"]["size"] - 被上传文件的大小，以字节计
$_FILES["file"]["tmp_name"] - 存储在服务器的文件的临时副本的名称
$_FILES["file"]["error"] - 由文件上传导致的错误代码

    strrchr() 函数查找字符串在另一个字符串中最后一次出现的位置，并返回从该位置到字符串结尾的所有字符。
    javascript部分
    indexOf：是查某个指定的字符串在字符串首次出现的位置;
    lastIndexOf：是查某个指定的字符串在字符串末尾出现的位置;
    stringObject.substring(start,stop)：substring() 方法用于提取字符串中介于两个指定下标之间的字符；

    0x02 客户端检测绕过(javascript)

    检测方式：
    打开http反向代理burpsuite,选择所需要上传的webshell文件，点击”上传”，此时burpsuite中没有拦截到任何消息，但是弹出一条警告框，此时则可以认为客户端javascript检测。javascript检测方式较为简单，此处暂且省略。绕过方式也有很多种，burpsuite拦截更改后缀，浏览器禁用javascript等。

    0x03 服务端检测绕过MIME

    此处只更改php中的代码，更改方式如下：

    $is_upload = false;
$msg = null;
define("UPLOAD_PATH", "../upload/");
if (isset($_POST['submit'])) {
    if (file_exists(UPLOAD_PATH)) {
        if (($_FILES['upload_file']['type'] == 'image/jpeg') || ($_FILES['upload_file']['type'] == 'image/png') || ($_FILES['upload_file']['type'] == 'image/gif')) {
            if (move_uploaded_file($_FILES['upload_file']['tmp_name'], UPLOAD_PATH . '/' . $_FILES['upload_file']['name'])) {
                $img_path = UPLOAD_PATH . $_FILES['upload_file']['name'];
                $is_upload = true;
            }
        } else {
            $msg = '文件类型不正确，请重新上传！';
        }
    } else {
        $msg = UPLOAD_PATH.'文件夹不存在,请手工创建！';
    }
}
?>

    if (($_FILES['upload_file']['type'] == 'image/jpeg') || ($_FILES['upload_file']['type'] == 'image/png') || ($_FILES['upload_file']['type'] == 'image/gif'))

    检测方式：
    打开http反向代理burpsuite,选择所需要上传的webshell文件，点击”上传”，此时burpsuite中可以发现拦截信息，将Content-Type值改为image/png后点击转发，即可上传成功，具体操作过程演示如下：

    0x04 服务端检测绕过

    服务端文件扩展名检测中，分为两种：基于黑名单的检测绕过，基于白名单的检测绕过；黑名单检测安全性相对于白名单低很多，绕过方式也多种多样：
    （1）文件名大小写绕过；
    （2）名单列表绕过；
    （3）特殊文件名绕过；例如：发送的http包中把文件名改为test.asp. 或者 test.asp_ 这种文件名命名方式在windows系统下是不被允许的，通过burpsuite进行修改，然后绕过验证后，会被windows系统自动去掉后面的点和空格，注意linux系统中没有这种特性；

    大小写绕过的方式

    在php增加部分代码：

     $deny_ext = array(".php",".php5",".php4",".php3",".php2",".html",".htm",".phtml",".pHp",".pHp5",".pHp4",".pHp3",".pHp2",".Html",".Htm",".pHtml",".jsp",".jspa",".jspx",".jsw",".jsv",".jspf",".jtml",".jSp",".jSpx",".jSpa",".jSw",".jSv",".jSpf",".jHtml",".asp",".aspx",".asa",".asax",".ascx",".ashx",".asmx",".cer",".aSp",".aSpx",".aSa",".aSax",".aScx",".aShx",".aSmx",".cEr",".sWf",".swf",".htaccess"); //危险文件后缀黑名单
$file_name = trim($_FILES['upload_file']['name']); //获取文件名
$file_name = deldot($file_name);//删除文件名末尾的点
$file_ext = strrchr($file_name, '.'); //获取后缀
$file_ext = str_ireplace('::$DATA', '', $file_ext);//去除字符串::$DATA
$file_ext = trim($file_ext); //首尾去空
if (!in_array($file_ext, $deny_ext))

    通过上面代码可以分析，是基于黑名单的过滤方式，分别进行了去除文件名的点、空格操作，随后匹配危险后缀名，但是缺少大小写转换，故此时可以通过大小写绕过。将文件后缀名改为.PHP .PHp .PhP等即可绕过。

    可解析文件后缀绕过方式

    在php增加部分代码：

    $deny_ext = array('.asp','.aspx','.php','.jsp'，'.htaccess');
$file_name = trim($_FILES['upload_file']['name']);
$file_name = deldot($file_name);//删除文件名末尾的点
$file_ext = strrchr($file_name, '.');
$file_ext = strtolower($file_ext); //转换为小写
$file_ext = str_ireplace('::$DATA', '', $file_ext);//去除字符串::$DATA
$file_ext = trim($file_ext); //收尾去空
if(!in_array($file_ext, $deny_ext))

    通过代码可以分析，是基于黑名单的过滤方式，分别进行了去除文件名的点、空格操作，随后匹配危险后缀名，此时危险后缀名并没有写全，此时可以根据黑名单过滤不全进行绕过：可以将后缀名改为 .php5 .php3等进行绕过。对于可解析的后缀名作如下总结：

    asp|asa|cer|cdx|aspx|ashx|ascx|asax
php|php2|php3|php4|php5|htaccess
jsp|jspx

    .htaccess绕过方式

    $deny_ext = array(".php",".php5",".php4",".php3",".php2","php1",".html",".htm",".phtml",".pHp",".pHp5",".pHp4",".pHp3",".pHp2","pHp1",".Html",".Htm",".pHtml",".jsp",".jspa",".jspx",".jsw",".jsv",".jspf",".jtml",".jSp",".jSpx",".jSpa",".jSw",".jSv",".jSpf",".jHtml",".asp",".aspx",".asa",".asax",".ascx",".ashx",".asmx",".cer",".aSp",".aSpx",".aSa",".aSax",".aScx",".aShx",".aSmx",".cEr",".sWf",".swf");

$file_ext = strrchr($file_name, '.');

    通过代码可以分析，是基于黑名单的过滤方式，分别进行了去除文件名的点、空格操作，随后匹配危险后缀名,几乎涵盖了所有的危险后缀名，但是发现没有.htaccess。故此时可以先上传.htaccess文件，此时就能将任意文件格式解析为php,再上传一个图片格式的木马即可；其.htaccess文件内容如下：

    SetHandler application/x-httpd-php

    0x07 安全防御措施

    基于白名单的过滤方式；
    对于文件进行重命名(随机数字，或者日期)或者将文件名转换成小写，去空格去点；
    攻击者不能自己创建目录；
    降低上传文件的执行权限；
    网站根目录增设.htaccess文件，限定可解析文件类型；
